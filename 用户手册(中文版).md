# DTU HPC 跳板 + 内网服务器（10.198.119.245）Tunnel（VS Code Remote-SSH 版）

目标：在 **VS Code** 里直接编辑/运行 `10.198.119.245` 上的代码（Remote-SSH 连接远端环境）。由于必须先登录 DTU HPC（login 节点）作为跳板，我们采用 **本地 SSH tunnel**：把远端 `10.198.119.245:22` 映射到本机 `localhost:2222`，让 VS Code 只连接 `localhost:2222`（也就是下面配置里的 `lab245-tunnel`）。

> 电脑重启/断网/睡眠会导致 tunnel 失效，需要重新启动。

## 0.1 你应该从哪一步开始（很重要）

很多人**已经**配置过免密登录（手里已经有 `~/.ssh/id_rsa` 或 `~/.ssh/id_ed25519` 之类的私钥），这时不需要重复生成/安装 key。

### A. 先测：你是否已经能免密登录 HPC？

在本机终端执行（把 `username` 换成你的 DTU 用户名）：

```bash
ssh username@login1.hpc.dtu.dk
```

- 如果你能直接登录进去，且**不需要输入 HPC 的账户密码**（可能会提示输入你本机私钥的 passphrase，或出现 DTU 的交互式认证/验证码，这些都算正常），那么：
  - **跳过第 2 节（给 HPC 装公钥）**
  - 第 1 节里“生成跳板 key”也可以跳过（除非你想新建一把独立的 key）。
- 如果提示 `Permission denied` / 一直要你输入 HPC 密码：继续按第 1、2 节做。

### B. 再测：你是否已经能免密登录 10.198.119.245（通过 HPC 跳板）？

在本机终端执行（同样替换 `username`）：

```bash
ssh -J username@login1.hpc.dtu.dk username@10.198.119.245
```

- 如果你能登录进去，且**不需要输入目标机的账户密码**：
  - **跳过第 3 节（给 10.198.119.245 装公钥）**
- 否则：继续按第 3 节做。

## 1) 准备 SSH key（可以“一把通用”，也可以“两把分开”）

> 你**不必须**让 “HPC 的 key” 和 “10.198.119.245 的 key” 不同。
>
> - **方案 1：复用同一把 key（最省事）**：你已经有 `id_rsa` / `id_ed25519`，就继续用它同时登录 HPC 和 10.198.119.245。
> - **方案 2：两把 key 分开（更推荐）**：一把专门给 HPC 跳板，一把专门给目标机。权限隔离更清晰。

先看看你本机已经有哪些 key：

```bash
ls -al ~/.ssh
```

如果你看到 `id_rsa` / `id_ed25519`（以及对应的 `.pub`），通常就可以直接复用。

### 1.1（可选）生成一把“跳板 key”（给 HPC 用）

如果你**已经能免密登录 HPC**，这一步可以跳过。

```bash
mkdir -p ~/.ssh
ssh-keygen -t ed25519 -f ~/.ssh/dtu_hpc
```

生成后：
- 私钥：`~/.ssh/dtu_hpc`
- 公钥：`~/.ssh/dtu_hpc.pub`

### 1.2（可选）生成一把“目标机 key”（给 10.198.119.245 用）

如果你**想和 HPC 分开**，可以单独生成：

```bash
ssh-keygen -t ed25519 -f ~/.ssh/lab245_key -C "username@10.198.119.245"
```

生成后：
- 私钥：`~/.ssh/lab245_key`
- 公钥：`~/.ssh/lab245_key.pub`

> 如果你打算“一把 key 走天下”，那第 1.2 节也可以跳过；后续把 `IdentityFile` 指向你现成的 `~/.ssh/id_rsa` 或 `~/.ssh/id_ed25519` 即可。

## 2) 把公钥装到 DTU HPC（让你免密登录跳板）

> 如果你在第 0.1A 的测试里已经能免密登录 HPC：**跳过本节，直接到第 3 / 第 4 节**。

### 2.1 选择要装到 HPC 的公钥文件

两种常见情况：

- 你用现成 key：例如 `~/.ssh/id_rsa.pub` 或 `~/.ssh/id_ed25519.pub`
- 你新建了跳板 key：`~/.ssh/dtu_hpc.pub`

下面用 `PUBKEY=...` 来表示你选中的公钥路径。

### 2.2 推荐：用 ssh-copy-id（macOS/Linux 通常有）

```bash
PUBKEY=~/.ssh/dtu_hpc.pub  # <- 改成你的 .pub
ssh-copy-id -i "$PUBKEY" username@login1.hpc.dtu.dk
```

### 2.3 手动方式（任何系统都可）

1) 先登录 HPC：

```bash
ssh username@login1.hpc.dtu.dk
```

2) 在 HPC 上执行：

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
cat >> ~/.ssh/authorized_keys
# 现在回到你本机另一个终端执行：cat <你的公钥文件>.pub
# 复制输出的那一整行（ssh-ed25519 / ssh-rsa ...），粘贴到上面这条命令的输入中，然后回车
# 最后按 Ctrl-D 结束输入
chmod 600 ~/.ssh/authorized_keys
```

## 3) 把公钥装到 10.198.119.245（让你免密登录目标机）

> 如果你在第 0.1B 的测试里已经能免密登录 10.198.119.245：**跳过本节，直接到第 4 节**。

### 3.1 先登录到 10.198.119.245

两种方式二选一：

**方式 A（推荐，从本机直连但经 HPC 跳板）：**

```bash

```

**方式 B（如果你已经在 HPC login 节点里）：**

```bash
ssh username@10.198.119.245
```

### 3.2 安装你的公钥到目标机

在目标机 `10.198.119.245` 上执行：

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
cat >> ~/.ssh/authorized_keys
# 回到本机执行：cat <你的公钥文件>.pub
# 复制那一整行，粘贴到这里，然后回车
# Ctrl-D 结束
chmod 600 ~/.ssh/authorized_keys
```

> 你可以把**同一把**公钥同时装到 HPC 和目标机，也可以分别装不同的公钥。

## 4) 配置本机 ~/.ssh/config（含 login/login1/login2 轮询）

编辑 `~/.ssh/config`，加入（把 `username` 和 `IdentityFile` 按你的情况改掉）：

```sshconfig
### --- DTU HPC jump hosts ---
Host dtu-login dtu-login1 dtu-login2
  User username
  # 下面 IdentityFile 可以指向你已有的 id_rsa/id_ed25519，
  # 也可以指向你新建的 ~/.ssh/dtu_hpc。
  IdentityFile ~/.ssh/dtu_hpc
  IdentitiesOnly yes
  ServerAliveInterval 30
  ServerAliveCountMax 3
  ConnectTimeout 8

Host dtu-login
  HostName login.hpc.dtu.dk

Host dtu-login1
  HostName login1.hpc.dtu.dk

Host dtu-login2
  HostName login2.hpc.dtu.dk

### --- target behind DTU (10.198.119.245) ---
Host lab245
  HostName 10.198.119.245
  User username
  # 这里也可以复用同一把 key（和上面 dtu-login 的 IdentityFile 一样），
  # 或者用单独的 ~/.ssh/lab245_key。
  IdentityFile ~/.ssh/lab245_key
  IdentitiesOnly yes
  ProxyCommand sh -c 'ssh -o ConnectTimeout=5 -W %h:%p dtu-login || ssh -o ConnectTimeout=5 -W %h:%p dtu-login1 || ssh -o ConnectTimeout=5 -W %h:%p dtu-login2'

### --- local tunnel endpoint (for VS Code Remote-SSH / testing) ---
Host lab245-tunnel
  HostName localhost
  Port 2222
  User username
  # 通常这里用“目标机的 key”（和 Host lab245 一致）
  IdentityFile ~/.ssh/lab245_key
  IdentitiesOnly yes
```

> 如果你使用 ssh-agent 且不想在 config 里写死 IdentityFile，也可以把 `IdentityFile ...` 那几行删除，让 OpenSSH 自己挑选 key。

验证：

```bash
ssh -v dtu-login1
ssh -v lab245
```

## 5) 建立本地 Tunnel

### 5.1 启动 tunnel（后台运行，自动尝试 login/login1/login2）

```bash
ssh -fN -o ExitOnForwardFailure=yes \
  -o ServerAliveInterval=30 -o ServerAliveCountMax=3 \
  -L 2222:10.198.119.245:22 dtu-login \
|| ssh -fN -o ExitOnForwardFailure=yes \
  -o ServerAliveInterval=30 -o ServerAliveCountMax=3 \
  -L 2222:10.198.119.245:22 dtu-login1 \
|| ssh -fN -o ExitOnForwardFailure=yes \
  -o ServerAliveInterval=30 -o ServerAliveCountMax=3 \
  -L 2222:10.198.119.245:22 dtu-login2
```

> `-fN` 会在认证成功后将进程放到后台；认证过程中输入的密码/验证码不会回显属正常。

### 5.2 检查 tunnel 是否有效

看本机 2222 端口是否在监听：

```bash
lsof -iTCP:2222 -sTCP:LISTEN -n -P
```

做一次 SSH 探活：

```bash
ssh lab245-tunnel 'echo TUNNEL_OK && hostname'
```

### 5.3 关闭 tunnel

```bash
lsof -ti tcp:2222 | xargs kill
```

## 7) 配置 VS Code（Remote-SSH，连接 localhost:2222）

> 只要你能在终端里成功执行 `ssh lab245-tunnel`（或 `ssh lab245`），VS Code 基本就能连上。

### 7.1 安装 VS Code 插件

在 VS Code 的扩展（Extensions）里安装：

- **Remote - SSH**（`ms-vscode-remote.remote-ssh`）
- **Python**（`ms-python.python`）
- （可选）**Jupyter**（`ms-toolsai.jupyter`），如果你要在远端跑 notebook

### 7.2 连接方式 A：通过本地 tunnel（本文主流程，推荐稳定）

1) 先确保 tunnel 已经启动（见第 5 节 / 第 6 节），并能探活：

```bash
ssh lab245-tunnel 'echo TUNNEL_OK && hostname'
```

2) VS Code 打开命令面板：

- macOS：`⌘ + Shift + P`
- Windows/Linux：`Ctrl + Shift + P`

输入并选择：

- `Remote-SSH: Connect to Host...`

然后选 **`lab245-tunnel`**（来自你 `~/.ssh/config` 的 `Host`）。

3) 第一次连接可能会让你选择远端平台（Linux / macOS / Windows），一般选 **Linux**。

4) 连接成功后，左下角会显示类似 `>< SSH: lab245-tunnel`。

### 7.3 连接方式 B：直接通过跳板机连接（可选，不需要本地 2222）

如果你不想维护本地 tunnel，VS Code 的 Remote-SSH 也可以直接读取第 4 节里 `lab245` 的 `ProxyCommand` 配置：

1) VS Code 命令面板 → `Remote-SSH: Connect to Host...`

2) 选择 **`lab245`**

### 7.4 打开远端目录、运行终端命令

连接成功后：

- `File → Open Folder...`，选择远端路径（例如 `/home/username/` 或你的项目目录）
- 打开 VS Code 终端（`Terminal → New Terminal`），它会在远端执行

你可以用这些命令确认确实在目标机上：

```bash
hostname
whoami
pwd
python3 -V
```

### 7.5 选择远端 Python 解释器

1) 在远端窗口里打开命令面板，运行：

- `Python: Select Interpreter`

2) 选择你要用的解释器（例如系统 `python3`、conda env、venv 等）。

如果你想在远端创建 venv（示例）：

```bash
python3 -m venv ~/venvs/lab245
source ~/venvs/lab245/bin/activate
python -m pip install -U pip
# pip install -r requirements.txt
```

然后在 `Python: Select Interpreter` 里选择：

- `~/venvs/lab245/bin/python`

### 7.6 常见问题（VS Code）

- **连不上 / 卡住**：先在终端确认 `ssh lab245-tunnel`（或 `ssh lab245`）能否正常登录；不行就先修 SSH / tunnel（第 1–6 节）。
- **提示 `Permission denied (publickey)`**：检查 `~/.ssh/config` 里 `IdentityFile` 是否正确，以及第 2/3 节是否把公钥放进了对应机器的 `~/.ssh/authorized_keys`。
- **远端安装 VS Code Server 失败（空间不足/权限问题）**：清理远端 `~/.vscode-server`，或确认 home 目录空间配额是否足够。

## 8) ⚠️ 警告：部分网络环境可能被 DTU/HPC 拒绝（Connection refused）

你可能会遇到如下现象：

- `nc -vz github.com 22` 成功
- 但 `nc -vz login1.hpc.dtu.dk 22` 返回 `Connection refused`

这通常表示 **DTU/HPC 边界策略对当前出口公网 IP/网段拒绝**。此时 tunnel / ssh 都会失败。

建议处理方式：
1) 连接 **DTU VPN**
2) 或切换网络（例如手机热点/其它 Wi‑Fi）
